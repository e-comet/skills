# Участие в разработке

Предлагайте изменения через pull request в ветку `main`.

Перед отправкой:

1. Оставляйте в PR только файлы, необходимые для публичного плагина.
2. Не добавляйте секреты, локальные пути, внутренние ссылки и персональные данные.
3. Опишите задачу, изменение поведения и способ проверки.

Принятые изменения проходят проверку и включаются в очередной снимок релиза. После публикации сопровождающий
оставляет в PR ссылку на версию и закрывает его как включенный в релиз. Поэтому принятый PR отображается как
закрытый, а не как слитый напрямую в `main`.
