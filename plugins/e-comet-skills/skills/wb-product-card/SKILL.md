# Карточка товара WB

## Контракт

Скилл получает данные карточек товаров Wildberries (одной или батчем до 50 артикулов) через
браузер пользователя: MCP tool `browser_job` подписывает задание, расширение e-Comet выполняет
запросы к WB, результаты читаются со страницы через RPC `window.__ecometAgent`. Сервер e-Comet
не выполняет запросы к WB и не видит ответы — все данные остаются в браузере пользователя.

## Требования

- **MCP сервер `e-comet`** — устанавливается автоматически вместе с плагином; при первом
  обращении откроется вход по email (код из письма).
- **Расширение e-Comet** — установлено в браузере, выполнен вход (введён ключ расширения).
- **Управляемый браузер** — агент должен уметь открывать вкладки и выполнять JavaScript на
  странице (page.evaluate или аналог). Без выполнения JS результаты прочитать нельзя.

## Как работает

1. Вызовите MCP tool `browser_job`:

   ```json
   { "job_type": "product_card", "articles": [791050753, 913357757] }
   ```

   До 50 артикулов, дубликаты схлопываются. Ответ: `trigger_url`, `job_ids` (один родительский
   `jobId`), `expires_at` (~5 минут — открывайте сразу, не откладывайте).

2. Сразу возьмите Chrome пользователя с расширением, а не встроенный (in-app) браузер агента:

   ```js
   agent.browsers.get("extension")
   ```

   Во встроенном браузере расширения e-Comet нет — задание там не выполнится.

3. Откройте `trigger_url` в новой вкладке этого браузера **как opaque-строку**: передайте
   значение ровно как вернул tool — не перепечатывайте, не пересобирайте, не редактируйте и
   не переносите через многострочный текст. Если внутри URL всё же оказались пробельные
   символы (`/\s/`) — значение повреждено при переносе, запросите новый `browser_job`.

4. Опрашивайте прогресс снипетом `scripts/agent_poll.js` (замените `__JOB_ID__` на
   `'<jobId>'`), раз в 2-3 секунды, до `progress.status === 'done'`.
   Если вернулся `setupError` — прекратите опрос и покажите пользователю `setupError.message`
   как есть (см. «Ошибки»).

5. Прочитайте результаты снипетом `scripts/agent_read.js`. Юниты джобы (`__UNIT_IDS__`):
   - `<jobId>__nm<nm>__detail` — ответ `cards/v4/detail`: `data.body.products` с ценой,
     рейтингом и остатками `sizes[].stocks[]`;
   - `<jobId>__nm<nm>__card` — контент `card.json`: склейка `colors`, описание,
     `grouped_options`;
   - `<jobId>__local` — локальные данные расширения: `storesData` (справочник складов
     `wh` → имя) и `productCardDetail` (кэш перехваченного detail).

   При батчах из многих артикулов читайте юниты порциями по 1-2 за вызов — полные тела большие.

## Как отвечать пользователю

- **Не оставляйте сырые имена полей WB в ответе** — переводите их на русский; булевы
  значения показывайте как «да/нет». Минимальный глоссарий:

  | Поле WB | В ответе |
  | --- | --- |
  | `imt_id` | Склейка (ID склейки) |
  | `subjectId` | ID предмета |
  | `vendorCode` / Vendor code | Артикул поставщика |
  | `need_kiz` | Обязательная маркировка (да/нет) |
  | `time1`, `time2` (в `stocks`) | Время доставки — одно число `time1 + time2`, не сырые поля |

- **Источник истины** по цене/рейтингу/остаткам — юнит `__detail`. Данные `__local`
  дополняющие и best-effort: на первой загрузке могут быть пустыми, деградируйте молча
  (например, показывайте `wh <id>` вместо имени склада).
- **Таблица «по складам»**: просуммируйте `qty` из `products[].sizes[].stocks[]` по `wh`,
  имена складов возьмите из `local.storesData[wh].name`; время доставки склада — сумма
  `time1 + time2`.
- **Таблица «по размерам»**: сумма `qty` внутри каждого элемента `sizes[]`.
- **Склейка**: артикулы из `colors` юнита `__card`.
- Юнит со `status: 'done'`, но `data.ok === false` / `data.status === 0` — сетевая ошибка WB:
  сообщите об этом по конкретному артикулу и предложите повторить.
- Указывайте, что остатки — моментальный снимок на момент запроса.

## Ошибки

`progress().setupError` — задание не принято расширением, покажите `message` как есть:

| `code` | Что сделать пользователю |
| --- | --- |
| `extension_not_authenticated` | Открыть попап расширения e-Comet и ввести ключ расширения |
| `extension_misconfigured` | Обновить/переустановить расширение |
| `token_rejected` | Запросить новый `browser_job` и повторить |

Ошибки уровня юнита (`status: 'error'`, поле `error`): `Agent job timed out` (нет ответа за
30 с), `Fetch URL host is not allowed`, `Extension connection not ready` / `closed`
(расширение недоступно — попросите проверить установку). Снипет-ответ
`{ error: 'ecomet-agent-rpc-not-available' }` означает, что расширение не запустилось на
вкладке — проверьте, что открыт именно `trigger_url` на wildberries.ru.
