# Фото товаров WB

## Контракт

Используйте этот скилл, чтобы получить URL фото для артикулов Wildberries. В разных системах эти значения
также могут называться `product_id`, `nm_id`, `nmId` или SKU.

Для поиска всегда используйте `scripts/wb_images.py`. Не собирайте итоговые списки ссылок вручную, если только скрипт
не сломан и пользователь явно не попросил восстановить результат вручную.

Скрипт проверяет URL-кандидаты и возвращает только фото, которые сейчас доступны.

## Быстрый старт

Запуск из корня публичного репозитория:

```bash
python3 plugins/e-comet-skills/skills/wb-product-images/scripts/wb_images.py 123456 12345678
```

Компактный отчет для чтения человеком:

```bash
python3 plugins/e-comet-skills/skills/wb-product-images/scripts/wb_images.py 123456 --max-photos 5 --format markdown
```

Если идентификаторы лежат в файле:

```bash
python3 plugins/e-comet-skills/skills/wb-product-images/scripts/wb_images.py --input-file product_ids.txt
```

Файл ввода может содержать идентификаторы, разделенные пробелами, переносами строк, запятыми или точками с запятой.

## Опции

- `--max-photos N`: проверить номера фото `1..N`; по умолчанию `15`.
- `--max-basket N`: проверить корзины от `basket-01` до `basket-N`; по умолчанию `60`.
- `--size big|tm`: выбрать размер фото WB; по умолчанию `big`.
- `--format json|markdown`: выбрать формат вывода; по умолчанию `json`.
- `--basket-config PATH`: необязательный JSON в формате ответа `misc/wb_baskets` из приложения. Если он передан,
  медиа-хост для volume артикула проверяется перед общим fallback-поиском по корзинам. Без конфига скрипт использует
  встроенную карту media-корзин и проверяет дополнительные корзины на случай новых диапазонов WB.

## Как отвечать пользователю

Верните вывод скрипта и кратко укажите:

- сколько артикулов удалось обработать;
- сколько артикулов не найдено;
- какие некорректные входные значения отклонил скрипт.

Не утверждайте, что отсутствующие артикулы не существуют на WB. Формулируйте это как "не найдены текущей проверкой
image-CDN".
